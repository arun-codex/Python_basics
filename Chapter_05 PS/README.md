# 🧩 Chapter 05 Practice Set: Dictionaries and Sets

This is the ultimate test of your Dictionary and Sets knowledge. Solve these without looking at the solutions.

## 📝 Problem Index

### Problem 1: Hindi to English Dictionary
- **Difficulty:** ⭐☆☆☆☆
- **Concept Tested:** Dictionary Initialization and `.get()` lookup.
- **Expected Time:** 5 Mins
- **Hint:** Ask for user input and pass that input into `.get()`.
- **Time Complexity:** $O(1)$
- **Space Complexity:** $O(1)$
- **Common Mistake:** Using `dict[input]` which crashes if the user types a word not in the dictionary.
- **Revision Priority:** Low (Basic syntax)

### Problem 2: Unique Numbers
- **Difficulty:** ⭐⭐☆☆☆
- **Concept Tested:** Set uniqueness.
- **Expected Time:** 10 Mins
- **Hint:** Take 8 numbers as input, put them in a list, then convert the list to a set.
- **Time Complexity:** $O(n)$
- **Space Complexity:** $O(n)$
- **Common Mistake:** Forgetting that sets are unordered and trying to print them in the order they were entered.
- **Revision Priority:** Medium

### Problem 3: Safe Dictionary Keys
- **Difficulty:** ⭐⭐⭐☆☆
- **Concept Tested:** Hashability.
- **Expected Time:** 5 Mins
- **Hint:** Can you use `s = {18, "18"}`? Can you use a list?
- **Expected Approach:** Integers and strings with the same visual value are distinct types. Lists are unhashable.
- **Follow-up Interview Question:** "Why are lists unhashable in Python?"
- **Revision Priority:** High (Interview Trap)
